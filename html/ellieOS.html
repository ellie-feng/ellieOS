üöß Challenges & Solutions

Challenge 1: Window Dragging Conflicts

When windows overlapped, dragging would move multiple windows simultaneously.

Solution

Implemented proper event listener cleanup and focus management. Only the focused window responds to drag events.

Challenge 2: Terminal Readability

Initial green-on-black terminal was hard to read and clashed with the light theme.

Solution

Redesigned with black text on light base color, using accent colors for different command types. Much more legible while maintaining terminal aesthetic.

Challenge 3: Window Sizing

Fixed-size windows wasted screen space on large displays.

Solution

Implemented CSS resize property and percentage-based initial sizes. Windows adapt to viewport while remaining manually resizable.

What I Learned

  • State management is complex: Managing window positions, focus, and z-index without a framework taught me deep JavaScript fundamentals
  • Details matter: Authentic retro styling required studying original Windows UI down to pixel-level border rendering
  • User testing reveals assumptions: Initially thought vim keybindings were intuitive‚Äîmost users needed hints
  • Performance considerations: Multiple active windows require careful event handling to avoid memory leaks
  • Accessibility is hard: Balancing heavy theming with screen reader support requires intentional design
